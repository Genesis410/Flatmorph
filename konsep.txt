# 4. Core Architecture

# Traditional VDOM

```text id="2z4a0i"
Component
 → JS Object Tree
 → Recursive Diff
 → Patch Queue
 → DOM
```

Masalah:

* object allocations banyak,
* recursive traversal,
* memory churn.

---

# nano-morph-vdom

```text id="t4qwbj"
HTML Template
 → Flat Token Array
 → Bitmask Diff
 → Direct DOM Patch
```

No recursive VDOM.

---

# 5. Technical Design

# A. Flat Lookup Array

Alih-alih:

```js
{
  type: "div",
  children: [...]
}
```

Kamu pakai:

```text id="2q0l2k"
[ID, TYPE, FLAGS, PTR]
```

Contoh:

```text id="avh5p8"
[12, DIV, 0101, ref]
[13, TEXT, 0010, ref]
```

Semua sequential memory.

Keuntungan:

* cache friendly,
* no nested traversal,
* no recursive GC nightmare.

---

# 6. Key Innovation

# Bitwise Mutation Mask

Setiap node punya:

```text id="0s7rto"
0001 = text changed
0010 = attribute changed
0100 = child changed
1000 = visibility changed
```

Diff jadi:

```js
if (mask & TEXT_CHANGED)
```

Bukan:

* deep compare,
* recursive compare,
* object identity traversal.

Ini MASSIVE performance gain di low-end CPU.

---

# 7. WeakRef DOM Binding

Alih-alih simpan object VDOM:

```text id="av3t4h"
Node ID → WeakRef(real DOM node)
```

Keuntungan:

* GC natural,
* no duplicated tree memory,
* direct patch access.
